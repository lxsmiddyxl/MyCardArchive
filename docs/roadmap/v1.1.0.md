# MyCardArchive v1.1.0 roadmap

Internal planning for the next milestone after **v1.0.0**. Dates are indicative; scope is ordered by dependency and risk.

## Themes

1. **Community v2** — richer read paths, safer write surfaces, clearer reporting outcomes, and optional curator workflows built on the Phase 63 feed v1 + Phase 69 moderation hooks.
2. **Marketplace v3** — deeper discovery, calmer notification cadence, and pricing transparency without turning the app into a storefront.
3. **Realtime v3** — fewer reconnect surprises, clearer degradation UX, and tighter presence semantics for clubs and trades.
4. **Mobile polish v3** — touch targets, offline-friendly shells, and PWA install flows aligned with collector habits.

## Milestones (draft)

| Milestone | Goal | Notes |
|-----------|------|--------|
| M1 | Community v2 read/write split | Ship curated read APIs; keep mutations narrow and rate-limited. |
| M2 | Marketplace v3 discovery | Build on existing market APIs; avoid scope creep into payments beyond Stripe today. |
| M3 | Realtime v3 reliability | Instrument reconnect budgets; align with health overlays already in staging. |
| M4 | Mobile polish v3 | Audit top journeys (binders, trades, scan) on small viewports. |

## Internal notes

- Prefer **small vertical slices** per milestone so CI (typecheck, lint, unit, Playwright smoke) stays green.
- **No schema change** unless a milestone explicitly includes a migration review.
- Keep **product analytics** (`product.*` events) aligned with any new surfaces so funnels stay interpretable.

## Success criteria for v1.1.0 tag

- Community and market **happy paths** covered by E2E smoke plus one dual-account trade scenario where applicable.
- **Moderation**: report flow exercised in staging; internal review route tested with `INTERNAL_TELEMETRY_SECRET`.
- **Performance**: `npm run scale:smoke` against staging documented in runbooks.
