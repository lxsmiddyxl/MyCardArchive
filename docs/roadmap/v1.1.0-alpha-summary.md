# MCA v1.1.0-alpha — ecosystem summary

This marker summarizes the **v1.1.0-alpha** arc: qualitative social and marketplace intelligence, safer community structure, richer presence, and profile depth — **without** real-money checkout flows.

## Analytics

- Product analytics hooks and surface telemetry continue to power funnel understanding.
- Activity waves combine platform rhythm with **zone-level heat** (presence v3) for ambient, non-exact signals.

## Retention loops

- Retention hints strips remain on key collection surfaces.
- Offline read-only banners on **binders** and **decks** lists clarify read-only behavior when the device is offline.

## Community v2

- Posts may carry an optional **`topic_slug`**, stored as a machine-readable first line (`[mca:topic:…]`) while keeping bodies plain text.
- `GET /api/community/posts` and `GET /api/community/feed/v1` accept a **`topic`** filter.
- Internal moderation review logging accepts an optional **`topic_key`** for channel-level audit metadata (no direct DB writes).

## Marketplace v3 foundation

- DTOs for **offers**, **listings**, and **qualitative price signals** live under `src/lib/dto/marketplace-v3.ts` with mappers in `src/lib/marketplace/v3-mappers.ts`.
- Read-only **`/api/market/v3/views`** snapshot plus **rate-limited** discovery routes for cards and trades.
- **Trend** endpoint: `GET /api/market/v3/trends?catalog_card_id=` returns relative interest labels (not prices).

## Reputation v3

- `trustScoreV3` blends graph reputation with **trade completion**, **community participation**, and a **bucketed report pressure** term — documented in `src/lib/reputation/reputation-v3.ts`.

## Presence v3

- Platform activity waves now include **`zone_heat`**: coarse buckets per collector room type (spark / gathering / steady / quiet) without coordinates.

## Creator tools v1

- Showcases support a **featured** marker (machine line in description), **notes** via `community_posts` with a stable prefix, and profile surfacing for the featured showcase.

## Social feed v3

- Supplemental feed rows for **showcase_created**, **trade_completed**, and **follow_edge_created** merge with SQL-backed feed events and rank under the same v4 stack.
